Opinionzoom
Ficou mais prático analisar um produto antes de comprá-lo. Com o OpinionZoom você não precisa mais ficar lendo centenas de comentários para decidir se o celular que você quer comprar vale a pena mesmo. Basta informar o nome do produto desejado para o programa buscar as opiniões sobre ele na web. Esses dados serão então processados utilizando técnicas de análise de sentimentos para gerar um resumo das opiniões muito mais prático de se interpretar. Essa aplicação foi desenvolvida no ICMC-USP como parte do projeto OPINANDO do NILC. Aqui você encontrará também as ferramentas que forem desenvolvidas pelo projeto OPINANDO disponíveis para utilização online. Resumo das opiniões

Como funciona?

O processo para se obter os resumos de opinião sobre um produto partindo de um conjunto de revisões foi denominado de pipeline de sentimento. Nesse processo são usadas diversas ferramentas disponibilizadas nesse site de forma a refinar e resumir os dados.

Pipeline de sentimento

Crawler de revisões

O produto informado será buscado em sites de e-commerce brasileiros de onde serão extraídas todas as revisões de usuários que forem encontradas. Atualmente a busca é feita apenas no site da Amazon e utiliza-se o primeiro produto resultante da busca para extrair as revisões.

Filtragem por qualidade

As revisões extraídas passarão por um filtro de qualidade, que irá analisar a formaçao de sentençcas de cada revisão e removerá da base as que aparentarem ter menor carga de sentimento.

Normalização textual

As revisões restantes passarão pelo normalizador Enelvo onde será feita a substituição de abreviações usadas na web e a correção de palavras erradas.

Filtragem por subjetividade

Após normalizadas, as revisões passam por um filtro de subjetividade que analisará sentença por sentença. Aquelas que não forem subjetivas, ou seja, que não puderem ser classificadas como tendo sentimento positivo ou negativo, serão removidas.

Identificação e classificação de aspectos

A próxima consistirá em identificar os aspectos citados nas revisões(um celular, por exemplo, as pessoas irão provavelmente falar sobre os aspectos bateria, câmera e tela) e classificar se estão falando bem ou mal sobre eles. Nessa etapa são gerados gráficos que permitem visualizar quantas vezes falou-se bem ou mal de cada aspecto.

Sumarização

Na última etapa será gerado um resumo textual sobre as opiniões do produto. O resumo levará em conta os aspectos classificados e poderá ser feito usando diferentes métodos. O tipo de sumarizador a ser usado pode ser modificado na opção "configuração" antes de iniciar o pipeline de sentimento. Consulte a aba "Ferramentas"->"Sumarizador de revisões" para saber mais sobre cada tipo de sumarizador disponível.

Ferramentas

As ferramentas disponibilizadas aqui foram desenvolvidas pelo OPINANDO, um projeto de mineração de opinião para o português pertencente Núcleo Interinstitucional de Linguística Computacional (NILC). Abaixo você encontrará uma breve explicação sobre cada ferramenta e sua respectiva referência.

Normalizador enelvo

Enelvo é uma ferramenta de normalização de CGU desenvolvida para o português. Ela é capaz de identificar e corrigir ruídos em textos da web, como tweets, reviews de produtos e posts em redes sociais. Os principais ruídos tratados são erros ortográficos, internetês, acrônimos, nomes próprios, entre outros.

Filtro de qualidade de revisão

O objetivo dessa ferramenta é analisar e estimar o grau de importância de comentários gerados por usuários da Web. Para isso ela verifica os aspectos reputação do usuário(quando possível), riqueza do vocabulário utilizado e frequência de uso de palavras opinativas.

Ela foi implementada utilizando dois métodos: Fuzzy e MLP. No primeiro foram designadas regras empiricamente combinando os 3 aspectos, enquanto que, no segundo, as regras foram aprendidas a partir de um corpus de revisões anotadas do buscapé. Para ambos a saída é dada para cada revisão classificando-a de 1 até 4(insuficiente a pleno) em ordem crescente de qualidade.

O método Fuzzy não está disponível para utilização online nesse site devido ao custo de processamento, no entanto pode ser usado no toolkit. O código fonte está disponível no github tanto para o método Fuzzy quanto para o MLP. Para informações mais detalhadas acesse os respectivos artigos aqui e aqui.

Filtro de subjetividade de sentenças

Essa ferramenta recebe como entrada um comentário gerado por um usuário da Web e faz análise de subjetividade de cada sentença usando método baseada em léxico. Nessa análise são contabilizadas as palavras de sentimento positivo("gostei", "feliz") e as de sentimento negativo("ruim", "decepção"), obtendo-se posteriormente a diferença entre as palavras positivas e negativas. Se essa diferença for diferente de 0, considera-se a sentença como subjetiva e ela permanece na revisão, caso contrário é removida.

O código está disponível no github do toolkit e é uma adaptação para python da implementação descrita pelo item 4 dessa publicação.

Ferreira, L.G.; Belisário, L.B.; Pardo, T.A.S. Classificação de subjetividade para a língua portuguesa.In the Proceedings of the VI Workshop de Iniciação Científica em Tecnologia da Informação e da Linguagem Humana, pp. 358-361. 2019.

Extrator e classificador de aspectos

Essa ferramenta permite encontrar aspectos do produto nos textos de revisão e, a partir das palavras de sentimento próximas, classificar se existe sentimento positivo ou negativo expressado com relação a eles.

Aspectos são palavras que descrevem alguma característica de determinada entidade(nesse caso produto). Um celular, por exemplo, terá os aspectos: bateria, preço, velocidade ou até mesmo o próprio produto. Nos textos de revisão eles podem aparecer de forma explícita(a palavra que define o aspecto aparece no texto) ou de forma implícita("")

Costa, R.W.M. Extração e qualificação de aspectos de opinião para o português. Trabalho de Conclusão de Curso. Instituto de Ciências Matemáticas e de Computação, Universidade de São Paulo. São Carlos-SP, November, 47p. 2018.
